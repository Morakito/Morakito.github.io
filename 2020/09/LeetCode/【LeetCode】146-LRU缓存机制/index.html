【LeetCode】146-LRU缓存机制

发表于 2020-09-09 | 分类于 LeetCode
字数统计: 887 | 阅读时长 ≈ 3

题目

分析

数据结构

LRU缓存机制可以通过哈希表（Hashtable）加上双向链表（Double Linked List）实现。

  • 双向链表按使用的现后顺序存储键值对，靠近尾部tail的键值对是最近使用的，靠近头部head的键值对是最久未使用的。
  • 哈希表使用unordered_map，存储key和双向链表结点的映射关系

具体思路

我们可以使用哈希表来进行定位，找到其在双向链表中的位置，再移动该结点到tail之前即可

  • 对于get操作，首先需要判断key是否存在
    • 若key不存在，则直接返回-1
    • 若key存在，则返回对应的键值，并将其移动到tail之前。
  • 对于put操作，首先也要判断key是否存在
    • 若key存在，则直接修改key对应的键值，并更新其位置
    • 若key不存在，则需要判断是否超出缓存的容量上限
      • 若未超出上限，直接将键值对放到tail之前，并加入哈希表
      • 若超出上限，则需要先删除head的下一个结点，然后再添加键值对

时间复杂度O（1）

访问哈希表的时间复杂度为O（1），在双向链表的头尾添加和删除结点的时间复杂度也是O（1）

空间复杂度O（n）

哈希表和双向链表最多存储n + 1个元素（n为缓存的容量上限）
